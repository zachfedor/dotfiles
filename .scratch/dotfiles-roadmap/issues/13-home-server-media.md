## A. Node management — facts & open items

**Roster:** hestia (mac), athena (NixOS desktop), mnemosyne (Synology NAS),
hephaestus (DO droplet), theseus (iPhone). New: the Pi.

- **Pi facts:** hostname `raspi`, older model, running **Raspbian**, static IP
  **192.168.1.2**. Jobs: always-on Tailscale node + **Pi-hole** network ad-block.
- **Rename → `argus` (DECIDED 2026-06-29).** `raspi` → **`argus`** (Argus
  Panoptes — never-sleeping, all-seeing guardian; fits always-on pi-hole/network
  node). Apply during the NixOS migration (hostname set in nix config).
- **NixOS migration → YES (DECIDED 2026-06-29).** Convert Pi off Raspbian to NixOS
  for declarative parity. Open sub-questions to settle at build: confirm arch
  (Pi 3/4? aarch64 vs armv7 — nixos-on-arm support + SD image), SD-card wear
  (consider `zramSwap`, log-to-RAM, or boot-from-USB), and **pi-hole in nix** —
  no first-class `services.pihole`; options = OCI container (`virtualisation.
  oci-containers` pihole/pihole image) vs native `services.dnsmasq` +
  `services.adguardhome` (AdGuard Home = cleaner nix story than Pi-hole, worth
  considering as the replacement). New `nixosConfigurations.argus` flake output +
  `hosts/argus/`. Likely its own slice.
- **`tailscale ssh` broken = host-key verification (DIAGNOSED).** NOT the
  `--ssh`-not-enabled case. Plain `ssh pi@192.168.1.2` still works → use that for
  now. Resolves on its own once the Pi is rebuilt as NixOS (fresh host keys +
  declarative tailscale). Don't chase it pre-migration.
- **Prompt bug on Pi over ssh (BUG, fix during migration).** Typing `ls` then
  backspacing leaves extra blank spaces — line-editing cursor math wrong. Same
  class as a prior **alacritty** prompt fix. Lead hypothesis: **TERM/terminfo
  mismatch** — alacritty sets `TERM=alacritty`, the Pi's older Raspbian terminfo
  DB lacks that entry → zsh ZLE miscounts width. Verify with `echo $TERM` on the
  Pi + `infocmp alacritty`. Fix: ship alacritty terminfo (NixOS: `pkgs.alacritty`
  / `environment.enableAllTerminfo = true`) or interim `TERM=xterm-256color ssh`.
  Secondary suspect: zsh prompt with non-printing escapes not wrapped in `%{…%}`.
  NixOS migration (shared `home.nix` zsh + proper terminfo) likely fixes it — add
  a verify step.
- **ssh config:** add `Host` matchBlocks (Tailscale names/IPs) for the NAS and Pi
  in `home.nix` programs.ssh, so `ssh mnemosyne` / `ssh argus` just work. Folds in
  the open loose end "reserve athena static IP + add `Host athena` matchBlock".

## Build progress

**Slice 3 (Jellyfin on mnemosyne) = DONE 2026-07-02.** Deployed via Container
Manager docker-compose project `jellyfin` (official `jellyfin/jellyfin` image,
`network_mode: host` → `:8096`, `user: "1026:100"`, config/cache under
`/volume1/docker/jellyfin`, `/volume1/Media Library` mounted `:ro`). Library
populated, playback verified, **Plex retired**.

**Synology-container ACL gotcha (reusable):** Docker `user: "1026:100"` runs the
process as uid 1026 + gid 100 (users) ONLY — Docker does NOT load the account's
supplementary groups, so the `group:administrators` ACE never matches even though
`nas` is in administrators. Symptom: `UnauthorizedAccessException` on `/config/log`
despite POSIX 777 (the `+` = Synology ACL overrides POSIX). Fix that worked: add an
explicit `user:nas` ACE. config/cache via `synoacltool -add … user:nas:allow:rwx…
:fd--`; **Media Library** via File Station → Properties → Permission → add `nas`
Read → "Apply to this folder, sub-folders and files" (recursive — `fd--` inherit
only covers NEW children, not Plex's existing per-file ACLs). Alt route (unused):
`group_add: [<admin-gid>]` in compose. Compose NOT committed to repo (NAS-side,
matches #06 syncthing precedent).
