# Experiment with Hyprland / keyboard-driven WM on athena

Status: needs-triage

## What to build

Experiment with **Hyprland** (or another keyboard-driven tiling WM) on **athena**
(NixOS desktop, AMD GPU, currently GNOME). Goal: a fast, keyboard-first window
management workflow with **parity to the Hammerspoon window keybinds** used on
hestia (mac), so muscle memory carries across both machines.

This was always planned as a separate issue from #08 ricing — #08 is theming
(base16/colors/fonts), this is the WM itself. They interact (a new WM needs its
bar/colors themed) but the decisions are distinct.

## Open questions (triage / grill)

- **Hyprland vs alternatives** — Hyprland (Wayland, modern, active) vs sway
  (Wayland, i3-compatible, simpler) vs i3 (X11, mature). athena is on Wayland
  (GNOME/GDM) already; Wayland-native (Hyprland/sway) is the natural fit and
  resolves the deferred **Wayland clipboard** loose end (wl-clipboard vs the
  X11-only xclip/xdotool currently installed).
- **Keybind parity** — enumerate the Hammerspoon window bindings on hestia, map
  each to a WM equivalent. Decide how much parity is worth chasing vs adopting
  WM-native idioms.
- **Coexist with GNOME** — keep GNOME installed as a fallback session (GDM
  session picker) during experimentation, or commit fully? Daily-driver stability
  matters — don't strand athena.
- **Scope of the rice** — bar (waybar?), launcher (wofi/rofi), notifications,
  lock, idle. How much to declare in nix now vs iterate.
- **Sequencing vs #08** — #08 establishes the base16 palette + fonts; a WM rice
  should consume that palette. Build #08 first, or co-design so the WM theme
  pulls from the same source (e.g. stylix, if #08 chooses it).

## Acceptance criteria

- [ ] WM chosen and installed declaratively on athena via the flake / home-manager
- [ ] GNOME fallback decision recorded (kept as session vs removed)
- [ ] Core Hammerspoon-parity window keybinds reproduced and documented
- [ ] Wayland clipboard working (closes the deferred xclip/xdotool loose end)
- [ ] Bar + launcher + lock present and themed (consumes #08 palette)
- [ ] Daily-driver path stays stable throughout
